Help / roadmap

v1 (this tool): compose + copy only. No auto-posting, no accounts, no network calls — safe to keep open all day. Drafts auto-save to this browser's localStorage (per file location); reopening restores them.

  • Workflow: type the Master message once → Push master to variants → trim each pane under its limit → Copy per platform.
  • The counter turns red and shows how far over you are once a pane exceeds its limit. LinkedIn's 3000 is a soft/style guide; Blog intro is unlimited.
  • Character counts are UTF-16 code units (what the textarea reports). X/Bluesky weight some things differently (URLs, emoji) — treat the counter as a close guide, not gospel.

Planned later (NOT in v1):

  • AI-assist: a button that drafts all the variants from the master (tone + length per platform) so you edit rather than trim by hand.
  • Actual scheduled posting: needs per-platform OAuth for X / Bluesky / Mastodon / LinkedIn — a much bigger, credential-holding project, deliberately out of scope here.

Not to be confused with: the automated release-broadcast living in the pdoom1-website repo (issue #144). That fires templated announcements on release; this tool is for hand-crafted, per-post content. Different jobs.
