P(Doom)1 -- Doom Overlay Preview

Demonstrates "doom is a layer, not a repaint": each base asset renders doom-neutral, and doom is composited on top as an ADDITIVE overlay (radial glow via globalCompositeOperation="lighter" + a low-alpha colour-grade + an edge aura). Drag the sliders -- the same untouched base pixels get progressively doomed. Turn every overlay off to see the raw base.

Axes

neutral -> green (ok) -> blue (acting up) -> purple (eldritch)
neutral -> amber -> red -> fire (hand-sourced)

Colour sourcing. Weirdness blue/violet stops are pulled from docs/art/palette.json (extracted hero palette: screen-blue #383761, doom-purple #351b33, doom-indigo #2e2547). The weirdness green "ok" stop is NOT in palette.json (the hero image has no green) -- it comes from the Axis-B table in docs/art/PALETTE_AND_DOOM_INTENSITY.md. The catastrophe amber/red/fire stops are hand-sourced: palette.json has no warm amber at all, so these are tasteful hand-picked values, not extracts from the hero. The 5 presets map to the doom BANDS (cosy / uneasy / spooky / eldritch / terminal) the game keys off (the same idea as a MUSIC_TIER_BY_BAND lookup): each band sets a (weirdness, catastrophe) pair.
