## Contributing to Documentation

When adding new documentation:

1. **Guides** - Implementation guides, how-tos, integration docs
2. **Summaries** - Development summaries, completion reports, phase retrospectives
3. **Analysis** - Technical analysis, investigation reports, research findings

Use clear, descriptive titles and place in the appropriate subdirectory.
