# Resolved and Reference Issues

This folder contains issue documentation that is **not actively being worked on** but is kept for historical reference and future debugging.

## Categories of Issues in This Folder:

### ✅ **Resolved/Completed Issues**
Issues that have been successfully implemented, fixed, or completed:
- Issues marked with "RESOLVED", "COMPLETED", or "FIXED" status
- Features that have been fully implemented and deployed
- Bugs that have been confirmed fixed

### 📚 **Reference Documentation**
Issues kept for future reference:
- Low priority documentation cleanup tasks
- Architectural discussions that may inform future decisions
- Investigation reports that could be useful for similar future issues
- Outdated issues that contain useful historical context

## Purpose

This organizational structure helps maintain:
- **Clean active issues list**: Only current, actionable items in the main issues folder
- **Historical preservation**: Important context and decisions are not lost
- **Future debugging**: Reference materials available when similar issues arise
- **Project continuity**: New contributors can understand past decisions and implementations

## When to Move Issues Here

Move issues to this folder when they are:
- ✅ **Completed/Resolved**: Implementation finished and verified
- 📋 **Low Priority**: Documentation cleanup or minor improvements that are not actively planned
- 🔍 **Reference Only**: Architectural discussions or investigations completed but kept for reference
- 📜 **Historical**: Outdated but potentially useful for understanding past decisions

## Folder Structure

```
issues/
├── [active issues - current work items]
└── resolved-and-reference/
    ├── README.md (this file)
    ├── [completed features and fixes]
    └── [reference documentation and investigations]
```
